Add **Send by Email** functionality to InvoiceBridge.

Goal:

Allow users to send the generated GST invoice PDF directly to a client, accountant, or finance team from the Download page.

This should complete the workflow:

Upload
→ Convert
→ Generate PDF
→ Download OR Email

---

# Tech Requirements

Use:

Resend

Integrate with existing Next.js backend.

Do NOT use SMTP or Nodemailer.

---

# Install Dependency

```bash id="5r9w1q"
npm install resend
```

---

# Environment Variables

Add:

```env id="1p7z4h"
RESEND_API_KEY=
EMAIL_FROM=noreply@invoicebridge.app
```

---

# Backend Implementation

## Create API Route

Create:

`src/app/api/send-email/route.ts`

Method:

POST

---

## Request Body

```json id="6m2s8c"
{
  "invoiceId": "string",
  "recipientEmail": "client@example.com",
  "subject": "Converted GST Invoice",
  "message": "Please find attached the converted GST invoice."
}
```

---

## Route Responsibilities

The API should:

### 1

Validate authenticated user

Reject if unauthenticated

---

### 2

Verify invoice ownership

Users can only email their own invoices

---

### 3

Fetch converted PDF from storage

Retrieve generated GST invoice

---

### 4

Send email with PDF attachment

Attach:

Converted GST invoice PDF

---

### 5

Return success response

```json id="2q8n5k"
{
  "success": true
}
```
