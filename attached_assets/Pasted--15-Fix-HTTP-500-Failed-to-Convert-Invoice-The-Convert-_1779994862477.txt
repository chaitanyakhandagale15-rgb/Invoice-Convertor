# 15. Fix HTTP 500 “Failed to Convert Invoice”

The Convert flow occasionally throws:

**HTTP 500 Internal Server Error: Failed to convert invoice**

Diagnose and permanently fix this.

Preserve all current functionality.

---

## Add Full Error Diagnostics

Update:

`src/app/api/convert/route.ts`

Wrap conversion logic in robust try/catch.

Log detailed structured errors:

```typescript
console.error("Convert API Error:", {
  message: error.message,
  stack: error.stack,
  invoiceId,
  requestBody
});
```

Return meaningful responses instead of generic 500.

Example:

```json
{
  "error": "Exchange rate fetch failed"
}
```

or

```json
{
  "error": "Invoice data extraction incomplete"
}
```

---

## Validate Incoming Payload

Before conversion, validate:

### Required fields

* invoiceId
* sourceCountry
* exchangeRate
* gstRate
* lineItems
* amount

Reject invalid requests with:

HTTP 400

Clear message.

## Handle Missing OCR Data Gracefully

Fallback safely.

Examples:

If amount missing:

Attempt derive from line items

USD → 83
GBP → 105
EUR → 90
CAD → 61
AUD → 55

Show warning toast:

Using fallback exchange rate

Conversion must continue.
